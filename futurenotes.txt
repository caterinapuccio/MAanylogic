[1] PROVIDER — WaitingForTruck state (deferred, medium priority)
────────────────────────────────────────────────────────────────────
CONTEXT:
  When Intermediate accepts a booking (IS path, acceptBooking returns true)
  but no truck is available (dedicated truck busy AND no jolly truck free),
  the current model accepts the booking, increments recyclingStoragePerMat,
  but dispatches nobody. The waste is "phantom-accepted": counted in storage
  but never physically collected.

CURRENT WORKAROUND (implemented):
  A cyclic event truckRetryEvent (every 0.25 day = 6h) polls for a free truck.
  When found, it dispatches and clears pendingWaste. This is functionally
  equivalent to a WaitingForTruck statechart state but implemented as a
  parallel event to avoid restructuring the existing single-thread statechart
  into an AND-composite state.

DESIRED BEHAVIOUR (future):
  If a proper parallel statechart is preferred for clarity, the Provider
  statechart could be refactored into an AND-composite state with:
    - Main thread: Idle → GeneratingWaste → IS_Accepted/IS_Rejected/NonIS_Disposal → Idle
    - Parallel thread: Idle ↔ WaitingForTruck (polls truckRetryEvent)

DESIGN REQUIREMENTS (if implementing as statechart instead of event):
  1. acceptBooking must be split into two steps:
       a. acceptCapacity(mat, qty, prov) — checks Tier1/Tier2, increments
          storage, returns true/false. Does NOT dispatch a truck.
       b. dispatchTruck(prov) — finds and sends a truck. Called from
          Provider or from a Intermediate cyclic event.
  2. Provider statechart: add WaitingForTruck state between IS_Accepted
     and Idle. From WaitingForTruck, poll periodically (e.g. every 6h)
     whether a truck is available. When found, dispatch and exit to Idle.
  3. Edge case: if a new weekly cycle fires while Provider is in
     WaitingForTruck, should the new waste be added to wasteOnHold or
     discarded? Decision needed before implementation.

AFFECTED FILES / CLASSES:
  - Provider statechart: add WaitingForTruck state and transitions
  - Intermediate.acceptBooking (or split into two methods)
  - GarbageTruck.home entry action (check pending queue)
  - Possibly: new variable wasteOnHold in Provider
  - Possibly: new ArrayList<Provider> pendingPickups in Intermediate

NOTE: This does not affect simulation correctness in the near term
because the jolly truck pool makes full-fleet-busy scenarios rare.
Prioritise after basic dispatch loop is verified working.

[3] ACCEPTBOOKING — silent no-dispatch bug
────────────────────────────────────────────────────────────────────
CONTEXT:
  In Intermediate.acceptBooking, when a booking is accepted (Tier1 or Tier2
  capacity available) but both the dedicated truck AND all jolly trucks are
  busy, the function logs "ACCEPTED", increments recyclingStoragePerMat,
  and returns true — but no truck is ever sent. The waste is counted in
  storage but never arrives physically. Over time this causes:
    - recyclingStoragePerMat to grow without bound
    - Eventually tier1Free = 0 for every material
    - All subsequent bookings rejected → all waste routes to NonIS_Disposal

  This is related to item [1] above (WaitingForTruck). Fix item [1] first.

  CURRENT MITIGATION: truckRetryEvent in Provider polls every 6h and
  dispatches when a truck becomes free. Watch [DISPATCH-FAIL-T1] /
  [DISPATCH-FAIL-T2] traceln tags to see how often this fires.

TRACELN TO WATCH:
  [DISPATCH-FAIL-T1] and [DISPATCH-FAIL-T2] in Intermediate.acceptBooking
  will fire whenever this silent failure occurs. Check console for these
  tags to quantify how often it happens before implementing the fix.

[5] TRUCK DISPATCH — RESOLVED (2026-03-31)
────────────────────────────────────────────────────────────────────
STATUS: FIXED — commits 917c28c, 59ea326, ebc586f

PROBLEM:
  Trucks never moved after dispatch. Root cause: send("Go", truck) between
  sibling agents (Intermediate → GarbageTruck) without AgentLink is silently
  dropped by AnyLogic. No error, no warning.

FIX APPLIED (3 commits):
  1. Replaced send("Go", truck) with direct field write: truck.dispatchPending = true
     at all 4 dispatch sites (Intermediate.acceptBooking Tier1/Tier2,
     Provider.truckRetryEvent, Provider.NonIS_Disposal).
  2. Added pollTimer (cyclic, 0.25 DAY) to GarbageTruck so AnyLogic
     re-evaluates the t_dispatch condition trigger periodically.
  3. Replaced getSpeed() with hardcoded 10.0 in transit entry diagnostic
     (getSpeed() requires CarExtension from Road Traffic Library; GIS agents
     using moveTo don't have it → RuntimeException).

VERIFICATION (10-year simulation):
  - truck0: 514 trips, 853.056t cumulative
  - Dispatch latency: ~0.08 days (within 6h pollTimer window)
  - Full cycle: DISPATCH-T1 → TRUCK0-DISPATCH → TRANSIT-ENTRY → LOADED-LEG
    → ARRIVED@BIWASE → HOME → provider notified → isTransporting cleared

REMAINING NOTE:
  pollTimer introduces up to 6h dispatch latency. If sub-hour dispatch is
  needed in the future, reduce the pollTimer period (e.g., 0.04 DAY = ~1h).
  Trade-off: more frequent polling = more CPU overhead per truck.

[9] FERROUS GRADING — HMS1 assumption (deferred, low priority)
────────────────────────────────────────────────────────────────────
CONTEXT:
  All FERROUS_METAL(2) flows are currently modeled as a homogeneous
  material without grading. The implicit assumption is HMS1 (Heavy
  Melting Scrap Grade 1: ≥6 mm thickness, clean of non-metallics).
  HMS1 is the dominant traded grade in Vietnamese industrial scrap
  markets and the standard reference grade in industrial ecology
  literature (Graedel et al. 2011; Daigo et al. 2017).

  A more granular model would distinguish:
    - HMS1 (heavy industrial plate/structural scrap)
    - HMS2 (lighter gauge, some coatings allowed)
    - Shredded scrap (lower value, mixed quality)
    - Cast iron (different composition, different EAF path)

  Each grade has a distinct market price and may require different
  pre-treatment (e.g. HMS2 needs decoating step).

DECISION NEEDED:
  If grading is implemented, add a `ferrousGrade` parameter to Provider
  (set from Excel), and apply a grade-dependent market price multiplier
  to revenue calculations. The LCA emission factor would also differ
  slightly (HMS1 → EAF direct; lower grades → additional pre-processing).

NOTE ON PRICE:
  HMS1 Vietnam/HCMC dealer price: ~280–320 USD/t (2022–2023, [TRAINING]).
  Primary source for Vietnam: Vietnam Scrap Metal Association (VASMA),
  HCMC scrap price boards (grey literature, [GREY]).
  Excel note: "Assumed grade: HMS1 equivalent [ASSUMED]" is in the
  Intermediate sheet, ferrous row, col E/F annotation.

IMPLEMENT AFTER: basic model is fully running and validated.

[10] NON-FERROUS MATERIAL DIFFERENTIATION (deferred, medium priority)
────────────────────────────────────────────────────────────────────
CONTEXT:
  NON_FERROUS_METAL(4) is a single material type in the current model,
  but the environmental significance of non-ferrous materials varies
  dramatically by sub-type. Al, Cu, Pb, and Zn each have distinct
  GWP, energy demand, and toxicity profiles.

  From the ESCAR (BIWASE, Nov 2022):
  - WEEE dismantling recovers Cu, Au, Ag, Al from circuit boards
  - Battery dismantling recovers Pb from lead-acid batteries
  - No quantified tonnages per non-ferrous sub-type are given

  For VSIP1's industrial tenant base (electronics, garments, light
  manufacturing), likely non-ferrous streams include:
    Al offcuts (packaging, structural), Cu wire scrap,
    Pb solder/battery, Zn coating scrap

  Differentiation is ENVIRONMENTALLY meaningful (Cu recycling saves
  ~6 t CO2-eq/t; Al saves ~9 t CO2-eq/t; Pb saves ~1.5 t CO2-eq/t).

DECISION NEEDED:
  Extend MaterialType enum to include ALUMINIUM(5), COPPER(6), LEAD(7),
  or keep NON_FERROUS as an aggregate. If extended, all 5-element arrays
  must become 8-element arrays and all material loops must be updated.
  Requires VSIP1-specific composition data.

IMPLEMENT AFTER: VSIP1 data is obtained from DONRE or VSIP Management.

[11] MSW PROVIDER — Binh Duong municipality (deferred, medium priority)
────────────────────────────────────────────────────────────────────
CONTEXT:
  BIWASE receives ~2,990 t/day domestic MSW province-wide (2022 EIA,
  Decision 1232/QD-BTNMT). No VSIP1-specific MSW volume exists in
  the ESCAR. A rough district-level proxy for Thuan An: ~2,324 t/week
  [ASSUMED].

  MSW composition (ESCAR p.37, operational data):
    Organic (biodegradable): ~43.2%
    Leachate:                ~6.0%
    Recyclables (plastic/metal/nylon): ~1.3%  → of which plastic ~2.84%
    Incineration/landfill:  ~49.5%

  KEY DISTINCTION — industrial vs. municipal plastic recyclability:
    Industrial plastic (single-source, uncontaminated): 85–95% recyclable
    Municipal plastic (mixed polymers, food-contaminated): 20–40% effective
    recyclability in SEA context (Wilson et al. 2015, Waste Management;
    Sarc et al. 2019, Waste Management). [LIT, Consensus verification pending]

  This means the conversionEffPerMat[0] currently used for all providers
  should NOT apply equally to an MSW provider.

DESIGN APPROACH (recommended hybrid):
  1. ABM: add 1 new Provider instance of subtype "MSW" with custom
     wasteCompositionVec[5] derived from ESCAR p.37 composition data.
  2. SD: model weekly MSW inflow with seasonal multiplier (dry/wet season
     Vietnam waste generation variation ~±15%, [TRAINING]).
  3. DES: assign separate recyclabilityFactor[m] to MSW provider,
     overriding conversionEffPerMat for that provider only.

IMPLEMENT AFTER: all existing providers are verified correct and
the VSIP1-specific data situation is clarified.

[12] FERROUS INFORMAL SECTOR — noIS emission factor (deferred, medium priority)
────────────────────────────────────────────────────────────────────
CONTEXT:
  In Vietnam and SEA, ~70–85% of industrial ferrous scrap is captured
  by informal itinerant buyers (*ve chai*) before formal waste collection
  (Nguyen et al. 2019, Resources Conservation & Recycling;
  Wilson et al. 2006, Habitat International). [LIT, Consensus pending]

  The IS environmental benefit for ferrous therefore lies NOT in
  waste diversion rate (already high in noIS pathway) but in:
    noIS path: informal induction furnace smelting → higher GWP,
               heavy metal emissions, dioxins/PM2.5
    IS path:   EAF formal recycling → ~0.4–0.6 kg CO2-eq/kg,
               certified quality output (ecoinvent 3.9: "steel
               production, electric, low-alloyed")

IMPLEMENTATION:
  Add `recyclingEmissionFactor[5][2]` — [material][IS=0/noIS=1].
  Apply different GWP factor per tonne recycled in IS vs. noIS path.
  This makes the IS benefit for ferrous appear in the GWP KPI, not
  in the landfill fraction.

REQUIRES: ecoinvent 3.9 value for informal smelting in SEA context.
Consensus verification of literature values needed before implementing.

[13] ASH FRACTIONS — MATERIAL-SPECIFIC + BRICK PRODUCTION (deferred, low priority)
────────────────────────────────────────────────────────────────────
CONTEXT:
  Current model uses bottomAsh_massFraction=0.25 and flyAsh_massFraction=0.04
  for ALL materials. In reality:
    Paper/organic: bottom ash ~20–30%, fly ash ~3–5% (close to current values)
    Plastic: bottom ash ~5–10%, fly ash higher + more hazardous (dioxins)
    Textiles: similar to paper
    Ferrous/non-ferrous: metals should NOT be incinerated; they pass through
      as slag residue, not combustion ash

  BIWASE brick production (ESCAR pp.14, 20, 70):
    Input: non-hazardous bottom ash + wastewater treatment sludge
    Capacity: 20,000–100,000 bricks/day (discrepancy between p.20 and p.70)
    This is an IS output stream from BIWASE and should be modeled as
    an outflow of the Intermediate agent with associated revenue.

IMPLEMENT AFTER: basic model validation complete.

[14] MATERIAL-SPECIFIC RECYCLING OPEX (deferred, medium priority)
────────────────────────────────────────────────────────────────────
CONTEXT:
  Current OPEX model uses a single storage-based rate:
    monthlyIntOpex = usedCapacity × opexPerStorageUnitPerMonth

  In reality, recycling OPEX varies significantly by material:
    Plastic shredding + washing: energy-intensive (~40–80 EUR/t)
    Paper baling: low OPEX (~10–20 EUR/t)
    Ferrous sorting: medium (~15–35 EUR/t)
    Non-ferrous sorting/segregation: high (~50–120 EUR/t, especially Cu)
    Textile shredding: medium (~25–50 EUR/t)

  Vietnam cost factor vs. Western Europe: 0.35–0.50 (lower labour/land).
  Sources: [TRAINING, Consensus verification pending when quota resets]

IMPLEMENTATION:
  Add recyclingOpexPerMat[5] array (SGD/t). In calculateOPEX, compute:
    materialOpex += recyc_processedPerMat[m] × recyclingOpexPerMat[m]
  Requires knowing monthly processed tonnage per material.

EXCEL: add recyclingOpexPerMat[5] row in Intermediate Investment sheet
with per-material values and citation column.
